# 단일 소스 맵 — 작성 가이드

하위 플랜에서 **literal로 재정의 금지**인 수치·enum·수식의 목록. 발견 시 "<문서> §<위치> 참조" 형태로 치환한다.

## 단일 소스: `.claude/project.json` 의 `singleSource`

단일 소스 맵은 **`.claude/project.json`의 `singleSource` 배열**에 프로젝트별로 정의한다. 이 파일에는 하드코딩하지 않는다.

```jsonc
// .claude/project.json
"singleSource": [
  {
    "item": "<식별자·수식·enum 이름>",
    "source": "<정본 위치, 예: docs/P07_Spec_planned.md §부록 X>",
    "refForm": "<하위 문서에서 쓸 참조 표기, 예: 'P07 §부록 X 참조'>",
    "detect": "<선택: 중복을 감지할 패턴, 예: 'class SomeEnum', 특정 상수값>"
  }
]
```

## 무엇을 단일 소스로 등재하나

여러 문서·코드에서 반복 인용되지만 **한 곳에서만 정의되어야 하는** 것:
- **enum·코드 목록** (상태값, 태그, 카테고리 등) — 한 곳에서 정의, 나머지는 참조
- **수식·상수** (계산식, 계수, 임계값, 범위) — 값이 여러 문서에 흩어지면 drift 발생
- **네이밍 식별자·용어** — 한 용어의 정의는 한 곳

## 감지·치환 알고리즘

단일 소스에 기재된 값이 하위 문서에 **literal로 재등장**하면 다음 순서로 치환 제안:

1. 해당 문단 전체를 "본 항목은 <source> 단일 소스. 본 문서 내 재정의 금지." 한 줄로 축약
2. 필요하면 구체값 1개만 참조 인용 (예: "수식은 <source>의 `<식>`을 사용한다.")
3. 변경 전후 diff를 사용자에게 제시하고 확인

**예외**: 사용자가 명시적으로 "단일 소스에 반영하되 하위에도 재기재하라" 요청 시에만 중복 허용.

## 비어 있을 때

`singleSource`가 비어 있으면 silent 중복 감지는 skip한다. 프로젝트에 반복 인용되는 정본 수치·enum이 생기면 그때 등재한다.
